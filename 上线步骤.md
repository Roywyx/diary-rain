# 上线步骤（GitHub Pages）

本项目是静态网页，可直接用 GitHub Pages 免费上线。

## 1. 推送到 GitHub

1. 在 GitHub 新建一个仓库。
2. 将当前项目上传到仓库根目录。
3. 确认默认分支为 `main`。

## 2. 开启 Pages

1. 打开仓库 `Settings` -> `Pages`。
2. 在 `Build and deployment` 中选择 `GitHub Actions`。
3. 等待 Actions 中 `Deploy to GitHub Pages` 执行完成。

## 3. 访问公开网址

1. 在 `Settings` -> `Pages` 页面查看站点地址。
2. 该地址可被任何人访问。

## 4. 后续更新

每次推送到 `main` 分支会自动重新部署。

## 5. Supabase 注意事项

1. `anon key` 可公开放前端（这是 Supabase 设计方式）。
2. 必须开启 RLS 并写策略，否则前端读写会报错。
3. 如出现跨域或权限错误，优先检查 RLS 策略。
